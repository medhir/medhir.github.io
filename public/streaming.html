Twitter's API provides extremely powerful tools for interacting with their services. In this blog post, I hope to show the power and creativity that can be harnessed through the use of their streaming API. This streaming service allows you to gather tweets equivalent to 1% of all public messages that go through Twitter's servers. The neat part of this API is that you can recieve tweets in real-time for specific users, keywords, URLs, and locations.

My plan is to use the streaming API to upload music to this box right above. The basic idea is to be able to tweet a link to a song I'm currently enjoying, and have it show up on my blog automatically.

At a high level:
  • I will open up a streaming connection to twitter to monitor my tweets.
  • The blog will be updated whenever I send a link to a SoundCloud song and the hashtag #changeSong
  • I will write a bash script to automatically change the link to the song.
